## ディレクトリ構成

```
website/
├── index.html        トップページ(リファレンス実装済み)
├── about.html        会社概要ページ(骨格のみ・本文は利用者さん制作)
├── services.html     事業内容ページ(骨格のみ・本文は利用者さん制作)
├── contact.html      お問い合わせページ(骨格のみ・本文は利用者さん制作)
├── css/
│   └── style.css     共通スタイルシート(設計トークン+コンポーネント)
├── assets/
│   └── images/       画像置き場(logo.png 等・F-LINX側で準備)
├── incoming/         利用者さん提出の生HTMLファイルを一時保管
├── README.md         このファイル
└── DESIGN_SYSTEM.md  デザインシステム詳細
```

## 利用者さんからの提出ファイルの取扱い

1. 利用者さんが Codex で生成した .html ファイルを `incoming/` フォルダに配置
2. ファイル名: `{ページ名}_{利用者名}_{日付}.html` 推奨(例: `about_yamada_20260520.html`)
3. Claude Code で内容を確認・統合(下記「統合チェックリスト」参照)
4. 統合完了したら、本来のファイル(`about.html` 等)を上書き
5. `incoming/` の原本は履歴として残す

### 統合チェックリスト
- [ ] ヘッダー・フッターが共通の構造になっているか(`index.html` 参照)
- [ ] ナビゲーションの `is-current` クラスが該当ページに付いているか
- [ ] `<link rel="stylesheet" href="css/style.css">` がheadに含まれているか
- [ ] 利用者さんが書いた `<style>` タグは可能な限り `style.css` に統合(個別CSSは最小限に)
- [ ] CSS変数(`--color-main` 等)を使っているか、ハードコード色を使っていないか
- [ ] `.container`, `.section`, `.card`, `.btn-primary` 等の共通クラスを使っているか
- [ ] レスポンシブ対応(768px以下で破綻していないか)
- [ ] 文章の誤字脱字、敬語の統一
- [ ] リンク切れがないか(全ページのナビ・フッターリンク)

### Claude Code に依頼する時のテンプレ
```
incoming/about_yamada_20260520.html を about.html に統合してください。
- ヘッダー・フッターは index.html と同じ構造に
- 共通CSSクラス(.section, .card, .heading-md 等)を使う
- インラインCSSは可能な限り style.css に移動
- DESIGN_SYSTEM.md のルールに従う
```
